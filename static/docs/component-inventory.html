🧭 Navigation Components

CoreNavigation

Mobile: Full Responsive Required on all pages

The 5-link navigation: Home | Library | Office | Tools | Help

Usage: Include in header component on every page
<nav class="core-nav">
  <a href="/home.html">Home</a>
  <a href="/library.html">Library</a>
  <a href="/office.html">Office</a>
  <a href="/tools.html">Tools</a>
  <a href="/help.html">Help</a>
</nav>

VaultDoorButton

Mobile: Full Required on all pages

Orange vault button in header - opens file picker for uploads

Usage: Place in header right side, before user menu
<button class="vault-door-btn" onclick="openVaultUpload()">
  🔒 <span>Vault</span>
</button>

RoleSwitcher

Mobile: Dropdown Admin only

Switch between roles for testing/access

Usage: Only in admin header or debug mode

BreadcrumbNav

Mobile: Hidden

Home > Office > Vault > Upload path indicator

Usage: Below header on sub-pages (hidden on mobile)

🎯 Mobile-First Guidelines

Touch Targets

All interactive elements must be minimum 44x44px (Apple) or 48x48px (Google)

Navigation Pattern

Mobile: Hamburger menu with drawer panel containing 5 core links + vault button

Vault on Mobile

Primary action is camera/gallery. Metadata capture simplified with smart defaults.

Desktop-Only Features

When mobile user hits a desktop-only feature, show: "This feature works best on desktop. Email yourself a link to continue?"
